Esta guía detalla los procedimientos necesarios para la creación de plugins administrados por MCART.

Requisitos previos

Esta guía supone que el usuario tiene los conocimientos suficientes de programación para comprender cómo funciona la herencia de interfaces y clases en .Net, además del uso de atributos.

Esqueleto básico de un plugin

Creación básica de plugins

  1. Cree un nuevo proyecto en la solución, o cree una solución nueva. El proyecto debe (idealmente, pero no necesariamente) ser de tipo Biblioteca de clases.

  2. Agregue las referencias necesarias para la aplicación de la cual este proyecto será un plugin, además de una referencia a MCART.

  3. Cree una nueva clase, la cual debe heredar el tipo Plugin.

  4. Agregue los atributos de identificación a la clase.

    La siguiente lista muestra una serie de atributos que pueden añadirse a la definición de la clase. Todos los atributos son opcionales, a excepción de TargetMCARTVersionAttribute.

    • NameAttribute

    • DescriptionAttribute

    • VersionAttribute

      Si se omite, se utilizará la versión del ensamblado.

    • BetaAttribute

      Agregue este atributo si el plugin es una versión beta.

    • AuthorAttribute

    • CopyrightAttribute

    • AuthorAttribute

    • LicenseFileAttribute, EmbeededLicenseAttribute o LicenseTextAttribute.

    • MinMCARTVersionAttribute

      Establece la versión mínima de MCART con la que este plugin es compatible.

    • TargetMCARTVersionAttribute

      Establece la versión de MCART para la cual este plugin está diseñado. Si se omite, el plugin no será cargado por la aplicación.

    • UnsecureAttribute

      Agregue este atributo si el plugin utiliza código no administrado (inseguro).

    • UnstableAttribute

      Agregue este atributo si el plugin contiene código inestable, por ejemplo durante sus fases iniciales de desarrollo.

  5. Defina el comportamiento de su plugin. (vea más detalles en las secciones a continuación)

Luego del siguiente procedimiento, ya cuenta con la estructura básica de un plugin de MCART. El siguiente ejemplo muestra el esqueleto vacío de un nuevo plugin:

C#
VB
Copy
/// <summary>
/// Esqueleto básico mínimo de un Plugin.
/// </summary>
[TargetMCARTVersion(0, 7, 0, 0)] // En este caso, el plugin se compilará para MCART 0.7.0.0
public class PluginExample : MCART.PluginSupport.Plugin
{

}
''' <summary>
''' Esqueleto básico mínimo de un Plugin.
''' </summary>
<TargetMCARTVersion(0, 7)> ' En este caso, el plugin se compilará para MCART 0.7.0.0
Public Class TestPlugin
    Inherits Plugin

End Class
Plugins genéricos

Un plugin genérico es un plugin que únicamente contiene código de interacción, por ejemplo, botones o menús que realicen funciones.

Agregando una interacción a un plugin

  1. Agregue un nuevo método a su clase. El método debe tener una firma compatible con EventHandler, aún si utiliza Windows Presentation Framework.

  2. Agregue el atributo InteractionItemAttribute a la definición del método.

  3. (Recomendado, opcional) agregue los atributos NameAttribute y DescriptionAttribute a la definición del método.

  4. Implemente su funcionalidad.

El plugin ahora contendrá un elemento de interacción, el cual se agregará a la lista de interacciones provista por PluginInteractions.

C#
VB
Copy
/// <summary>
/// <see cref="InteractionItem"/> de prueba.
/// </summary>
/// <param name="sender">Objeto que ha generado el evento.</param>
/// <param name="e">Argumentos del evento.</param>
[InteractionItem]
public void Test(object sender, EventArgs e)
{
    System.Diagnostics.Debug.Print("Prueba de plugin.");
}
''' <summary>
''' <see cref="InteractionItem"/> de prueba.
''' </summary>
''' <param name="sender">Objeto que ha generado el evento.</param>
''' <param name="e">Argumentos del evento.</param>
<InteractionItem>
Public Sub Test(ByVal sender As Object, ByVal e As EventArgs)
    System.Diagnostics.Debug.Print("Prueba de plugin.")
End Sub

Si el plugin necesita actualizar la lista de elementos de interacción, la clase base Plugin provee del campo protegido uiMenu el cual es una lista observable de elementos de interacción.

Agregando interacciones durante la ejecución del programa

  • Cree una nueva instancia de la clase InteractionItem, y agréguela a la colección uiMenu:

    C#
    VB
    Copy
    uiMenu.Add(new InteractionItem((a, b) => About(this), $"Acerca de {Name}"));
    uiMenu.Add(New InteractionItem(
    Sub(a, b)
        About(this)
    End Sub, $"Acerca de {Name}"))
